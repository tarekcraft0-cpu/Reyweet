نشر دائم لصفحة الهبوط (Retweet)
================================

النشر التلقائي (موصى به) — reyweet.vercel.app
----------------------------------------------
1) Vercel → Project → Settings → Git → اربط مستودع GitHub (Reyweet).
2) Root Directory = . (جذر المستودع) — يستخدم vercel.json تلقائياً.
3) كل push إلى فرع main يعيد النشر تلقائياً.

أو عبر GitHub Actions (إذا لم يعمل ربط Git):
  أضف Secrets: VERCEL_TOKEN, VERCEL_ORG_ID, VERCEL_PROJECT_ID
  الملف: .github/workflows/vercel-landing.yml

من جهازك بعد التعديل:
  git add landing vercel.json scripts
  git commit -m "تحديث صفحة التحميل"
  git push origin main

1) Netlify (الأسهل — بدون Git)
   - شغّل: npm run build
   - افتح: https://app.netlify.com/drop
   - اسحب مجلد landing بالكامل إلى الصفحة
   - سيعطونك رابط https://....netlify.app

2) Cloudflare Pages
   - npm run build
   - لوحة Cloudflare → Workers & Pages → Create → Upload assets
   - ارفع محتويات مجلد landing بعد البناء

3) GitHub Pages (مجلد landing = جذر المستودع)
   - ادفع هذا المجلد إلى مستودع جديد على GitHub
   - Settings → Pages → Build: GitHub Actions
   - ادفع على فرع main ليعمل workflow في .github/workflows/pages.yml

4) Render
   - من جذر المشروع الكبير: New Blueprint واختر render.yaml
   - أو Static Site يدوياً: Root = landing، Build = npm install && npm run build

ملاحظة: dist/styles.css يُنشأ بـ npm run build قبل أي رفع.
