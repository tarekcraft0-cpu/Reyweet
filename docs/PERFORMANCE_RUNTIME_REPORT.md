# Performance Runtime Report

_Generated by `renderProfiler.ts` — activate with `localStorage.retweet_perf=1`_

## How to measure

```js
localStorage.setItem('retweet_perf', '1')
// reload app
// use app 10 minutes
// click "Report" in PerfHUD or:
window.__retweetPerfReport
// mark user actions manually:
window.retweetMarkAction('send-message')
```

## Instrumentation added

| System | File | Behavior |
|--------|------|----------|
| Render count | `useRenderCount` / `useProfiledRender` | Every major screen/component |
| Slow render (>16ms) | `useProfiledRender` | `console.warn [perf][slow-render]` |
| Action burst (>3 renders) | `markPerfUserAction` | `console.warn [perf][action]` |
| Long tasks (>50ms) | `PerformanceObserver` | `console.warn [perf][longtask]` |
| Memory | 10s interval | `memorySamples[]` |
| Leak registry | intervals/listeners/observers | `getLeakRegistry()` |
| Auto report | every 10 min | `emitPerfReport('10min')` |

## Architecture changes (patches)

### 1. No root `useApp()` in App.tsx

- `useCurrentUser`, `useIsGuestSelector`, `useAppTheme`, `useAppLanguage`
- `useAccountSwitching`, `useAccountSessionKey`, `useUnreadMessageCount`
- `useUnreadNotificationCount`, `useAppActions` only

### 2. Granular store subscriptions

- `StoreApiCtx` + `useAppSelector` (`useSyncExternalStore`)
- `AppUiCtx` for non-state UI flags
- `appHooks.ts`: `useChats`, `usePosts`, `useReelsPosts`, etc.

### 3. Feed isolation

- `homeFeedPosts` patch in-place on like/comment (no full re-sort)
- `homeFeedSig` unchanged on likes/notifications
- `PostCard` / `HomeScreen` use selectors

### 4. Chat isolation

- `ChatScreen` → `useChats()` only (not full state)
- `ChatRoom` → actions + `lang`/`users`/`stickers` selectors
- Unchanged chat rows keep same object ref → memo-friendly

### 5. Media pause

- `VideoPauseWhenHidden` pauses video + audio offscreen
- `ReelMediaPlayer` pauses when `active=false`
- `StoryViewer` pauses via `useStoryPlayback` + `middleHold`
